### Redefining the Programming Paradigm

Traditional software engineering relies on explicit instruction. An engineer writes specific logic constraints ($If/Then$ rules) to map known inputs to desired outputs. Machine learning completely flips this architectural loop. 

In a machine learning pipeline, we feed the system raw data inputs and their corresponding historical output labels. The algorithm then uses computational optimization to dynamically deduce the underlying mathematical function that connects them.

#### The Core Operational Components

Every machine learning system relies on three fundamental blocks:

1. **The Representation Model**: The structural framework used to process inputs. This ranges from simple linear boundary weights ($y = mx + c$) to complex deep neural networks with billions of parameters.
2. **The Evaluation (Loss) Function**: A mathematical feedback loop (such as Mean Squared Error or Cross-Entropy Loss) that measures how far the system's current guesses are from the true real-world labels.
3. **The Optimization Engine**: The algorithm (like Gradient Descent) that systematically alters internal model parameters to minimize the loss score, slowly nudging the system toward high accuracy.

#### Supervised vs. Unsupervised Systems

* **Supervised Learning**: The model trains on labeled target data matrices to solve classification tasks (e.g., separating spam) or regression tasks (e.g., predicting asset prices).
* **Unsupervised Learning**: The system parses unlabeled raw data to discover hidden patterns or cluster matching data features entirely on its own (e.g., customer market segmentation).
